# Shanghai 类公开数据接入说明

## 一、文档目的

很多校园异常检测公开数据并不是直接提供 mp4，而是提供帧序列、索引文件和 `.npy` 标签。本项目的视频任务接口接收的是视频路径，因此需要先把帧序列整理成 mp4，再接入现有流水线。

这份文档同时说明公开数据的来源标注方式，便于在演示和材料中保持清晰的数据口径。

## 二、数据来源口径

如果使用 ShanghaiTech Campus 或同类公开数据，材料中建议包含：

1. 数据集正式名称。
2. 论文或项目来源。
3. 下载地址。
4. LICENSE 或使用限制。
5. 本项目中的用途，例如“用于算法验证和流程演示”。

公开数据与自采园区视频应保持区分，避免影响演示材料的数据可信度。

## 三、常见目录结构

以 `SHANGHAI_Test` 这类数据为例，常见文件包括：

| 路径/文件 | 说明 |
|------|------|
| `SHANGHAI_test.txt` | 片段索引，通常包含片段路径、帧数、异常标记和异常区间 |
| `frames/` | 每个片段对应的一组有序图片 |
| `label/` | 帧级标签，通常为 `.npy` 文件 |

不同版本的数据结构可能不完全一致，最终以实际下载包中的说明为准。

## 四、接入方式

当前接口 `POST /jobs/run_local_path` 面向视频文件路径：

- 可以传 `.mp4`
- 帧目录需先转换为视频文件
- `.npy` 标签可用于离线对照，不作为在线告警输入

推荐流程：

1. 从索引文件中选一个片段。
2. 找到对应的帧序列目录。
3. 使用 `ffmpeg` 转成 mp4。
4. 把 mp4 放入 `data/videos/` 或其他后端能访问的路径。
5. 按 `TESTING.md` 提交视频任务。

示例命令：

```bash
ffmpeg -framerate 25 -i frame_%06d.jpg -c:v libx264 -pix_fmt yuv420p demo_shanghai.mp4
```

路径示例：

- 本机：`C:/.../smartpark-security-platform/data/videos/demo_shanghai.mp4`
- Docker：`/app/data/videos/demo_shanghai.mp4`

## 五、关于 `.npy` 标签

当前后端没有实现 `.npy` 标签直接驱动告警。`.npy` 可以用于离线评估或人工对照；在线告警仍以视频任务输出为准。

如果后续要做离线评测，可以新增单独脚本读取 `.npy`，将其与任务输出的告警时间段做对比。该能力需要明确标注为离线评测，而不是在线告警逻辑。

## 六、演示说明

本项目可将 ShanghaiTech Campus 等公开校园场景数据整理成 mp4，用于验证轨迹分析和告警流程。

不同来源的数据适合不同用途：

- `RepCount`、`LLSP`：健身或重复动作先验数据，适合算法验证。
- ShanghaiTech Campus：公开校园监控异常检测数据，适合场景演示，但仍需写清来源。
- 自采园区视频：需要说明采集和使用授权。

## 七、数据管理建议

- 原始数据体积通常较大，建议放在 `data/external/` 或服务器数据盘。
- Git 中只保留必要的小样例和说明文档。
- README、测试文档和答辩材料中的数据口径要保持一致。
- 数据集名称、论文年份和授权条款应以公开来源为准。
